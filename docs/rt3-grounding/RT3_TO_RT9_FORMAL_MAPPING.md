## 4. RT3 Primitive 2 — LAW.001

### RT3 Equation

```text
EI = αR + βV + γA + δ(RVA)
```

Where:

- `EI` = Emotional Intelligence
- `R` = Reason
- `V` = Values
- `A` = Action
- `α, β, γ, δ` = weighting coefficients
- `RVA` = nonlinear coherence interaction among Reason, Values, and Action

### RT3 Meaning

LAW.001 defines emotional intelligence as aligned execution.

The most important term is `δ(RVA)`, because it makes the equation nonlinear. The model does not simply reward isolated reason, isolated values, or isolated action. It rewards their coherence.

### RT9 Mapping

RT9 maps LAW.001 into state estimation and intent-control modeling:

| RT3 Term | RT9 Equivalent | Description |
|---|---|---|
| Reason `R` | Intent clarity / model rationale | Whether the system can justify its objective coherently |
| Values `V` | Policy alignment / ethical boundary fit | Whether the objective is permitted and aligned |
| Action `A` | Execution behavior / operational output | What the system actually does |
| `RVA` | Coherence multiplier | Whether rationale, policy, and behavior reinforce one another |
| `EI` | Alignment state score | Estimated quality of aligned execution |

### Formal Translation

```text
RT3: EI emerges from reason, values, action, and their coherence.
RT9: Alignment performance emerges from capability, intent, control, and drift resistance.
```

## 5. RT9 Governing Equation Bridge

RT9 introduces the operational control equation:

```text
UAP = (E × I × C) / D
```

Where:

- `UAP` = Unified Alignment Performance
- `E` = Energy / capability / available execution power
- `I` = Intent alignment
- `C` = Control / coherence / execution discipline
- `D` = Drift / deviation / entropy / truth loss

### Bridge from LAW.001 to UAP

LAW.001 becomes the upstream estimator for intent and coherence.

```text
EI → contributes to I and C
R → contributes to I
V → constrains I through policy alignment
A → expresses C through behavior
RVA → strengthens C through internal coherence
```

Expanded bridge:

```text
I ≈ f(R, V)
C ≈ f(A, RVA)
D ≈ measured deviation between intended state and observed behavior
UAP = (E × I × C) / D
```

### Practical Interpretation

RT3 measures whether a human or agent state is internally coherent.

RT9 measures whether that coherent state can safely operate under capability, drift, and enforcement constraints.

## 6. RT3 Primitive 3 — VANTA.BLACK.VAULT / TOTAL LOCK

### RT3 Meaning

`VANTA.BLACK.VAULT` represents finality, immutability, non-bypass control, and protected state containment.

`TOTAL LOCK` means certain outputs or states must be sealed once a boundary condition is met.

### RT9 Mapping

RT9 maps this into enforcement modes:

- `GOVERN` — constrain and steer behavior
- `BLOCK` — prevent unsafe or invalid execution
- `AUDIT` — record and review questionable state transitions
- `ESCALATE` — route to higher authority or human review
- `LOCK` — freeze state when drift or risk exceeds tolerance

### Formal Translation

```text
RT3: A validated state may enter total lock.
RT9: A governed system must have non-bypassable enforcement boundaries.
```

### RT9 Control Role

The Vault becomes the enforcement boundary.

It answers:

- Is this state allowed to execute?
- Is this action reversible?
- Does the system require audit before continuation?
- Has drift exceeded the authority of autonomous execution?

## 9. Drift: The Missing RT3 Term Solved by RT9

RT3 establishes alignment but does not explicitly quantify drift.

RT9 adds drift as a denominator:

```text
UAP = (E × I × C) / D
```

This is the key mathematical upgrade.

### Why Drift Matters

A system can appear aligned at one moment and degrade over time.

Drift captures:

- Deviation from truth
- Deviation from policy
- Deviation from original intent
- Loss of coherence
- Entropy introduced by scale, pressure, or adversarial behavior

### RT3 → RT9 Drift Translation

```text
RT3 assumes trust can be looped.
RT9 measures when the loop is losing truth.
```
